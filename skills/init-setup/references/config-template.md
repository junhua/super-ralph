# Config Template

Authoritative template for `.claude/super-ralph-config.md`. Rendered in Step 4 of the init-setup workflow.

## How to render

For each `<detected>` placeholder below, substitute the value detected by `detection-procedures.md`. Use empty backticks `` ` ` `` for values that couldn't be detected (blank). Use the literal string `none` for project-board and frontend fields when they don't apply.

## Output file

Write the rendered template to `.claude/super-ralph-config.md` (relative to the repo root).

## Substitution rules

- Replace every `<detected>` with the detected value wrapped in backticks. If the value is itself a path like `core/backend/config-service`, the resulting line should look like: `- `$BE_DIR`: `core/backend/config-service``
- Replace `<detected or none>` similarly, but use the literal string `none` (wrapped in backticks) when the section isn't applicable (e.g., no project board, no frontend).
- Replace `YYYY-MM-DD` with today's date in ISO format.
- Do NOT add, remove, or reorder fields — downstream skills grep for these keys by exact match.
